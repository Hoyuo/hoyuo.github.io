1. 부록 Multi Module Architecture Guide

작성일 2025-10-05

TikTok Clone 프로젝트의 멀티 모듈 아키텍처 구조와 Melos 기반 모노레포 관리 가이드

🏗️ 아키텍처 개요

이 프로젝트는 기능 기반 모듈화 아키텍처와 Melos 모노레포 관리를 사용하여 확장 가능하고 유지보수 가능한 구조를 구축합니다.

핵심 원칙

  1. 독립적인 기능 모듈: 각 기능은 자체 pubspec.yaml을 가진 별도의 Flutter 패키지
  2. 공유 디자인 시스템: 중앙화된 디자인 토큰으로 일관성 유지
  3. 명확한 의존성 방향: 기능 → 공통 모듈 (역방향 의존성 없음)
  4. 모노레포 관리: Melos를 통한 통합 패키지 관리

✅ 개발 가이드라인

DO ✅

  1. 디자인 시스템 사용

  2. 명확한 모듈 분리

    • 각 기능은 독립적인 패키지
    • screens/, widgets/ 디렉토리 구조 유지

  3. 상대 경로 의존성

  4. publish_to: ‘none’

    • 모든 모듈 패키지에 명시

DON’T ❌

  1. 하드코딩된 값 사용 금지

  2. 역방향 의존성 생성 금지

  3. 기능 간 직접 의존성 최소화

    • 가능한 메인 앱을 통한 오케스트레이션

🚀 주요 이점

1. 독립성

  • 각 기능 모듈은 독립적으로 개발/테스트 가능
  • 다른 기능에 영향 없이 수정 가능

2. 재사용성

  • 디자인 시스템을 통한 일관된 UI
  • 위젯 재사용으로 중복 코드 최소화

3. 확장성

  • 새로운 기능 추가 시 기존 코드 영향 최소화
  • 모노레포 구조로 통합 관리 용이

4. 유지보수성

  • 명확한 책임 분리
  • 의존성 방향이 명확하여 리팩토링 안전
