7. 구조화된 출력 - QuizGPT 만들기

작성일 2025-10-06

🎯 이 챕터에서 배울 것

  • LLM이 생성하는 텍스트를 우리가 원하는 정확한 형식(JSON)으로 제어하는 방법
  • WikipediaRetriever를 사용하여 외부 지식 소스에서 동적으로 콘텐츠를 가져오는 방법
  • Pydantic/JSON Output Parser: 프롬프트 엔지니어링과 파서를 통해 LLM의 출력을 구조화하는 기법
  • Function Calling: 모델이 특정 함수의 인자(argument) 형식에 맞춰 JSON을 생성하도록 강제하는 더 강력하고 안정적인 기법
  • 두 개의 체인(퀴즈 생성, 채점)을 연결하여 복합적인 애플리케이션을 구축하는 방법
  • Streamlit의 st.form을 사용하여 여러 개의 질문을 한 번에 제출하고 처리하는 방법

Output Parsers: LLM 길들이기

🎯 이번 단계에서 배울 것

  • LLM이 생성한 평범한 텍스트를 엄격한 JSON 형식으로 변환하는 2단계 체인(생성 -> 포맷팅) 설계
  • questions_prompt: 주어진 컨텍스트를 기반으로 특정 형식의 텍스트 퀴즈를 생성하도록 지시
  • formatting_prompt: 생성된 텍스트 퀴즈를 JSON 형식으로 변환하도록 지시
  • JsonOutputParser: LLM이 생성한 JSON 문자열을 Python 딕셔너리로 파싱하는 방법

📝 1단계: 2단계 체인과 JSON 파서 구현

전체 코드 (pages/03_QuizGPT.py):

🔍 코드 상세 설명

1. 왜 2단계 체인을 사용하는가?
LLM에게 한 번의 프롬프트로 복잡한 작업(내용 기반 질문 생성 + 완벽한 JSON 형식 준수)을 모두 시키면 실패할 확률이 높습니다. 작업을 두 단계로 나누면 각 단계의 역할이 명확해져 훨씬 안정적인 결과를 얻을 수 있습니다.

  • questions_chain: 내용에 집중하여 퀴즈 질문과 답변을 텍스트로 생성합니다.
  • formatting_chain: 앞서 생성된 텍스트를 입력으로 받아, 오직 JSON 형식으로 변환하는 데에만 집중합니다.

2. JsonOutputParser

  • LLM은 종종 JSON 응답을 ```json ... ``` 코드 블록으로 감싸서 반환합니다. 이 파서는 parse 메소드에서 이러한 불필요한 문자열을 제거하고 json.loads()를 사용하여 순수한 JSON 문자열을 Python 객체(딕셔너리/리스트)로 변환합니다.

🔍 코드 상세 설명

1. st.form

  • with st.form(...) 블록 안에 있는 모든 위젯들은 하나의 폼으로 그룹화됩니다. 사용자가 여러 개의 라디오 버튼이나 텍스트 입력을 변경해도 앱은 재실행되지 않습니다.
  • 오직 st.form_submit_button()을 클릭했을 때만 스크립트가 재실행되며, 그 시점의 모든 위젯 값들이 한 번에 제출됩니다. 이는 퀴즈나 설문조사와 같이 여러 입력을 한 번에 처리해야 할 때 매우 유용합니다.

2. 채점 로직

  • st.radio는 사용자가 선택한 답변의 문자열을 value에 저장합니다.
  • {"answer": value, "correct": True} in question["answers"] 코드는 사용자가 선택한 답변이 정답 리스트 안에 있는지 확인하는 간결하고 효과적인 방법입니다.

🔍 코드 상세 설명

1. Function Calling이란?
Function Calling은 LLM에게 단순히 텍스트로 답변하는 대신, 우리가 정의한 특정 함수의 인자(arguments) 형식에 맞는 JSON 객체를 생성하도록 요청하는 기능입니다. 이는 LLM의 출력 형식을 매우 안정적으로 제어할 수 있는 강력한 방법입니다.

  • 장점: 복잡한 프롬프트 엔지니어링(예: 2단계 체인)이나 출력 파서 없이도, 모델이 거의 항상 완벽한 JSON 스키마를 준수하도록 강제할 수 있습니다.

2. .bind(function_call=...)

  • LCEL의 .bind() 메소드를 사용하여 LLM을 호출할 때 특정 함수를 호출하도록 설정할 수 있습니다.
  • function_call={"name": "create_quiz"}: LLM에게 create_quiz 함수의 인자 형식에 맞춰 응답하라고 강제합니다.
  • functions=[function]: LLM에게 create_quiz 함수의 스키마(구조)를 알려줍니다.

3. 결과

  • 체인을 실행하면, response의 content는 비어있고, additional_kwargs["function_call"]["arguments"] 안에 원하는 JSON 형식의 문자열이 담겨 반환됩니다.

  • .bind()를 사용하여 LLM에 함수 스키마와 함수 호출을 강제했나요?
  • response.additional_kwargs에서 생성된 JSON 결과를 성공적으로 추출했나요?
