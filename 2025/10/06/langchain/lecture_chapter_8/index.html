8. 웹사이트 전체를 아는 챗봇, SiteGPT

작성일 2025-10-06

🎯 이 챕터에서 배울 것

  • SitemapLoader를 사용하여 웹사이트의 사이트맵(sitemap.xml)을 읽고 모든 페이지의 URL을 자동으로 수집하는 방법
  • BeautifulSoup과 파싱 함수를 사용하여 스크래핑한 HTML에서 불필요한 부분(헤더, 푸터 등)을 제거하고 순수 텍스트만 추출하는 방법
  • Map-ReRank 체인: 검색된 여러 문서 각각에 대해 답변과 점수를 생성(Map)한 후, 가장 점수가 높은 답변을 선택하여 최종 답변을 구성(ReRank)하는 고급 RAG 기법
  • LCEL을 사용하여 Map-ReRank 체인을 직접 구현하는 방법
  • Streamlit을 사용하여 SiteGPT의 전체 UI와 상호작용 로직을 완성하는 방법

🔍 코드 상세 설명

1. SitemapLoader

  • AsyncChromiumLoader가 단일 페이지만 로드하는 반면, SitemapLoader는 웹사이트의 sitemap.xml 파일을 분석하여 사이트 내의 모든 페이지 URL 목록을 가져옵니다. 그 후 각 URL을 방문하여 콘텐츠를 스크래핑합니다.
  • 왜 사용하는가?: 단 하나의 URL(sitemap.xml)만으로 웹사이트 전체를 학습 데이터로 만들 수 있어 매우 효율적입니다.
  • filter_urls: 정규식을 사용하여 원하는 URL 패턴(예: 블로그 포스트만)을 가진 페이지만 스크래핑하도록 필터링할 수 있습니다.

2. 사용자 정의 파싱 함수 (parsing_function)

  • SitemapLoader는 각 페이지의 HTML 콘텐츠를 BeautifulSoup 객체(soup)로 변환하여 parsing_function에 전달합니다.
  • 이 함수 내에서 soup.find(...).decompose()와 같은 메소드를 사용하여 광고, 네비게이션 바, 푸터 등 답변 생성에 방해가 되는 불필요한 HTML 요소를 제거할 수 있습니다. 이를 통해 더 깨끗하고 관련성 높은 텍스트만 추출하여 LLM에 제공할 수 있습니다.

  • 대상 웹사이트의 sitemap.xml URL을 찾았나요? (보통 https://example.com/sitemap.xml)
  • SitemapLoader를 사용하여 사이트맵을 로드하고, parsing_function으로 HTML을 정리했나요?

Map-ReRank 체인 구현하기

🎯 이번 단계에서 배울 것

  • Map-ReRank 체인의 동작 원리를 이해합니다.
  • Map 단계: 검색된 각 문서에 대해 개별적으로 답변과 점수를 생성하는 체인을 만듭니다.
  • ReRank (Reduce) 단계: 생성된 여러 답변과 점수를 보고, 가장 신뢰도 높은 최종 답변을 선택(또는 종합)하는 체인을 만듭니다.

📝 1단계: Map-ReRank 로직 구현

전체 코드 (pages/04_SiteGPT.py):

🔍 코드 상세 설명

1. Map-ReRank의 동작 원리
stuff 방식이 모든 문서를 한 번에 컨텍스트에 넣는 것과 달리, Map-ReRank는 더 정교한 접근 방식을 사용합니다.

  • Map: 검색된 문서가 5개라면, 5개의 문서 각각에 대해 answers_prompt를 실행하여 5개의 독립적인 답변과 점수를 생성합니다. 이는 각 문서가 질문에 얼마나 관련이 있는지를 평가하는 과정입니다.

  • ReRank (Reduce): 생성된 5개의 답변+점수 텍스트를 모두 모아 choose_prompt의 컨텍스트({answers})로 전달합니다. 그리고 LLM에게 이 중에서 가장 점수가 높고 최신 정보인 것을 바탕으로 최종 답변을 선택하거나 종합하라고 지시합니다.

  • 왜 사용하는가?: stuff 방식보다 더 많은 문서를 처리할 수 있으며, 각 문서의 관련성을 개별적으로 평가하므로 최종 답변의 정확성과 신뢰도가 높아집니다. LLM이 여러 소스를 비교하고 대조하여 가장 좋은 답변을 찾도록 유도할 수 있습니다.

✅ 체크리스트

  • Map 단계에 해당하는 answers_prompt와 get_answers 함수를 작성했나요?
  • ReRank 단계에 해당하는 choose_prompt와 choose_answer 함수를 작성했나요?
  • LCEL을 사용하여 retriever -> get_answers -> choose_answer 순서로 전체 체인을 올바르게 연결했나요?

전체 코드 (pages/04_SiteGPT.py):

🔍 코드 상세 설명

이제 모든 조각이 맞춰졌습니다.

  1. 사용자가 사이드바에 sitemap.xml URL을 입력합니다.
  2. load_website(url) 함수가 호출되어 웹사이트 전체를 로드하고, 분할하고, 임베딩하여 retriever를 생성합니다. 이 과정은 @st.cache_data 덕분에 URL당 한 번만 실행됩니다.
  3. 사용자가 st.text_input에 질문(query)을 입력합니다.
  4. 질문이 입력되면, 우리가 구성한 Map-ReRank 체인이 invoke(query)로 실행됩니다.
  5. 체인의 최종 결과(result.content)가 st.markdown을 통해 사용자에게 표시됩니다.
