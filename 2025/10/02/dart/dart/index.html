1. Hello, Dart

1.1 Dart란 무엇인가?

  • Dart 는 구글(Google)이 개발한 현대적이고 다목적 프로그래밍 언어입니다.
  • 2011년에 처음 공개되었으며, 특히 Flutter 프레임워크의 기반 언어로 널리 알려져 있습니다.
  • 웹, 모바일, 데스크톱, 서버, 임베디드 시스템 등 다양한 플랫폼에서 실행 가능한 애플리케이션을 만들 수 있습니다.

Dart의 핵심 철학:

  • 생산성(Productivity): 간결하고 읽기 쉬운 문법으로 빠른 개발이 가능합니다.
  • 이식성(Portability): 한 번 작성한 코드를 여러 플랫폼에서 실행할 수 있습니다.
  • 성능(Performance): AOT(Ahead-Of-Time) 컴파일을 통해 네이티브 코드 수준의 빠른 실행 속도를 제공합니다.

1.2 Dart의 주요 특징

1. Null Safety (Null 안전성)

  • Dart 2.12부터 도입된 Sound Null Safety 는 null 참조 에러를 컴파일 시점에 방지합니다.
  • 변수가 null을 가질 수 있는지 명확히 구분하여, 런타임 오류를 크게 줄입니다.

2. AOT & JIT 컴파일

  • JIT (Just-In-Time): 개발 중 빠른 Hot Reload를 제공하여 생산성을 높입니다.
  • AOT (Ahead-Of-Time): 배포 시 네이티브 기계어 코드로 컴파일하여 빠른 실행 속도를 제공합니다.
  • 두 가지 컴파일 방식의 장점을 모두 활용할 수 있습니다.

3. 완전한 객체 지향 언어

  • 모든 것이 객체입니다. 숫자, 함수, 심지어 null도 객체로 취급됩니다.

2. Expression, Variables, Constants

2.1 주석 (Comments)

  • 주석(Comment)은 코드에 대한 설명을 작성하는 공간으로, Dart 컴파일러는 이 부분을 무시하므로 실제 프로그램 동작에는 영향을 주지 않습니다.
  • 복잡한 로직을 설명하거나, 코드의 의도를 명확히 하거나, 임시로 코드를 비활성화하는 등 다양한 용도로 사용됩니다.

주석의 종류:

  1. 한 줄 주석 (Single-line Comments): //

    • // 기호부터 해당 줄의 끝까지를 주석으로 처리합니다.

  2. 여러 줄 주석 (Multi-line Comments): /* ... */

    • /*와 */ 사이의 모든 내용을 주석으로 처리합니다. 여러 줄에 걸친 설명을 작성할 때 유용합니다.

  3. 문서화 주석 (Documentation Comments): /// 또는 /** ... */

    • 코드에 대한 공식적인 문서를 생성하는 데 사용되는 특별한 주석입니다.
    • 다른 개발자가 내 코드를 쉽게 이해하고 사용할 수 있도록 도와주며, IDE에서 함수의 설명 등을 툴팁으로 보여주는 기능을 합니다.

  • 변수(Variable) 는 데이터를 저장하기 위한 이름이 붙은 공간입니다. Dart에서 변수를 선언할 때는 일반적으로 var 키워드나 명시적인 타입을 사용합니다.

2.4 상수: final과 const

  • Dart에서는 한 번 값을 할당하면 변경할 수 없는 ‘상수’를 만들기 위해 final과 const 두 가지 키워드를 사용합니다. 두 키워드 모두 변수의 재할당을 막지만, 값이 결정되는 시점에 중요한 차이가 있습니다.

  • final (런타임 상수)

    • final로 선언된 변수는 최초 한 번만 값을 할당할 수 있습니다.
    • 값의 할당은 런타임(프로그램이 실행되는 시점)에 이루어질 수 있습니다. 즉, 실행 시점에 계산되는 값이나 함수의 반환값 등을 할당할 수 있습니다.

  • const (컴파일 타임 상수)

    • const는 final의 특징을 모두 가지면서, 추가적으로 컴파일 타임(프로그램이 컴파일되는 시점)에 값이 반드시 결정되어야 한다는 제약이 있습니다.
    • 따라서 const 변수에는 숫자, 문자열 리터럴과 같은 고정된 값만 할당할 수 있으며, DateTime.now()처럼 실행 시점에 결과가 달라지는 함수 등은 사용할 수 없습니다.

final vs. const 핵심 요약

특징 final const
재할당 불가능 불가능
값 결정 시점 런타임 (Runtime) 컴파일 타임 (Compile-time)
사용 예시 API 응답 결과, DateTime.now() 앱 버전, 고정된 설정값
  • 권장 사항: 어떤 값을 상수로 만들고 싶을 때, 가능하다면 항상 const를 먼저 고려하는 것이 좋습니다. const는 컴파일러가 코드를 최적화하여 성능을 향상시키는 데 도움을 줍니다. const를 사용할 수 없는 상황(런타임에 값이 결정되는 경우)에 final을 사용하세요.

2.5 변수 명명 규칙 (Naming Conventions)

3.1 문자열 타입 (String)

  • 이번 챕터에서는 Dart의 여러 기본 데이터 타입 중, 텍스트 데이터를 다루는 String 타입에 대해 자세히 알아봅니다.
  • String은 하나 이상의 문자가 연속된 형태로, 작은따옴표(') 또는 큰따옴표(")를 사용하여 값을 만듭니다.
  • Dart에서는 일관성을 위해 한 가지 따옴표 스타일만 사용하는 것을 권장합니다. (예: 작은따옴표로 통일)
  • String 타입은 이후 섹션에서 배울 문자열 연결, 여러 줄 문자열 등 다양한 기능을 제공합니다.

3.2 문자열의 내부 구조

  • Dart의 String은 UTF-16 코드 유닛(code units) 의 시퀀스로 표현됩니다.
  • 대부분의 일반적인 문자(알파벳, 숫자, 한글 등)는 하나의 코드 유닛으로 표현되지만, 일부 특수 문자나 이모지(emoji)는 두 개 이상의 코드 유닛이 조합되어 하나의 문자를 나타내기도 합니다.
  • 유니코드(Unicode) 는 전 세계의 모든 문자를 일관되게 표현하기 위한 국제 표준 규약이며, UTF-16은 이를 인코딩하는 방식 중 하나입니다.
  • Dart가 유니코드를 지원하기 때문에, 우리는 String 안에 다양한 언어의 문자와 이모지를 자유롭게 담을 수 있습니다.
  • 핵심: string.length가 항상 우리가 눈으로 보는 글자 수와 일치하지 않을 수 있다는 점을 기억하는 것이 중요합니다. 이는 특히 이모지나 일부 특수 문자를 다룰 때 주의해야 합니다.

3.3 문자열 기본 사용법

1. 따옴표 사용

  • Dart에서는 작은따옴표(')와 큰따옴표(")를 모두 사용하여 문자열을 만들 수 있습니다.
  • 한 종류의 따옴표로 시작했으면, 같은 종류의 따옴표로 끝나야 합니다.
  • 문자열 안에 다른 종류의 따옴표를 포함할 수 있습니다.
  • 같은 종류의 따옴표를 문자열 안에 포함하려면, 백슬래시(\)를 사용하여 이스케이프(escape)해야 합니다.

2. 문자열 연결 (Concatenation)

  • + 연산자를 사용하여 여러 문자열을 하나로 합칠 수 있습니다.
  • 또는, 코드에서 문자열 리터럴을 연달아 쓰면 컴파일러가 자동으로 하나의 문자열로 합쳐줍니다.

3. 문자열 보간 (Interpolation)

  • 문자열 안에 변수나 표현식의 값을 포함시키는 가장 효율적이고 권장되는 방법입니다.
  • $ 기호를 사용하여 변수나 표현식의 값을 문자열에 삽입할 수 있습니다.
  • $변수명: 변수의 값을 바로 삽입합니다.
  • ${표현식}: 복잡한 표현식이나 메서드 호출의 결과를 삽입할 때는 중괄호({})를 사용합니다.
  • 변수 하나만 사용할 때는 불필요한 중괄호를 생략하는 것이 코드 가독성에 좋습니다.

3.4 여러 줄 문자열과 특수 문자

1. 여러 줄 문자열 (Multi-line Strings)

  • 여러 줄에 걸친 긴 텍스트를 문자열로 만들려면, 따옴표 세 개(''' 또는 """)를 사용합니다.
  • 따옴표 세 개로 감싸진 공간 안에서는 줄바꿈(엔터)이 그대로 문자열에 반영됩니다.

2. 특수 문자 (Escape Characters)

  • 문자열 안에는 직접 입력하기 어려운 특수 문자들을 표현하기 위해 이스케이프 시퀀스(escape sequence) 를 사용합니다. 이는 백슬래시(\)와 특정 문자를 조합하여 만듭니다.

3.5 Object vs Dynamic

  • Object와 dynamic은 다양한 타입을 다룰 수 있다는 공통점이 있지만, 중요한 차이가 있습니다.

  • Object:

    • Dart의 모든 클래스가 상속받는 최상위 클래스입니다.
    • Object 타입의 변수에는 어떤 타입의 값이든 할당할 수 있습니다.
    • 하지만, 변수의 메서드나 속성을 사용하려면, 컴파일러가 타입을 알 수 있도록 타입 캐스팅(as)을 하거나 타입 체크(is)를 해야 합니다. 이는 타입 안정성을 보장해줍니다.

  • dynamic:

    • dynamic으로 선언된 변수는 컴파일 시점에 타입 검사를 하지 않습니다.
    • 어떤 타입의 값이든 할당할 수 있으며, 타입 검사 없이 메서드나 속성을 호출할 수 있습니다.
    • 만약 런타임에 해당 메서드나 속성이 존재하지 않으면, 에러가 발생하여 앱이 중단될 수 있습니다.
    • 따라서 dynamic은 타입 정보를 정말 알 수 없는 특별한 경우에만 제한적으로 사용해야 하며, 사용 시 매우 주의가 필요합니다.

  • 권장 사항: 코드의 안정성과 예측 가능성을 위해, 가능한 한 dynamic 대신 명확한 타입을 사용하거나 Object와 타입 체크를 함께 사용하는 것이 좋습니다.

4. Control Flow

4.1 Boolean

  • bool 타입은 논리적인 참(true) 또는 거짓(false) 값을 저장하는 데이터 타입입니다.
  • bool 타입을 사용하는 이유:
    • 명확성: 코드의 의도를 명확하게 표현할 수 있습니다. 예를 들어, if (isLoggedIn)은 if (userStatus == 1) 보다 ‘로그인 상태인지 확인한다’는 의미를 훨씬 명확하게 전달합니다.
    • 안정성: 다른 타입(예: 숫자 0 또는 1)을 사용하여 참/거짓을 표현할 경우, 프로그래머마다 다른 기준을 사용하게 되어 혼란을 야기하고 버그를 발생시킬 수 있습니다. bool 타입은 true와 false 두 가지 값만 허용하므로 코드의 일관성과 안정성을 높여줍니다.

4.2 Boolean 연산자 (Boolean Operators)

  • Boolean 연산자는 bool 값을 비교하거나 조합하여 새로운 bool 결과를 만들어내는 데 사용됩니다. if문과 같은 조건문에서 핵심적인 역할을 합니다.

< 보다 작음 5 < 3 false
>= 크거나 같음 5 >= 5 true

! NOT bool 값의 결과를 반대로 뒤집습니다. true는 false로, false는 true로 바꿉니다. !isTired

4.5 변수 스코프 (Variable Scope)

  • 변수 스코프(Variable Scope) 는 변수가 코드의 어느 부분에서 접근 가능한지를 정의하는 규칙입니다.
  • Dart에서는 중괄호({})로 둘러싸인 코드 블록이 하나의 스코프를 형성합니다.
  • 변수는 자신이 선언된 스코프 내에서만 유효하며, 해당 스코프를 벗어나면 접근할 수 없습니다.

스코프 규칙:

  1. 상위 스코프에서 하위 스코프로의 접근 : 상위 스코프에서 선언된 변수는 하위 스코프에서 접근할 수 있습니다.
  2. 하위 스코프에서 상위 스코프로의 접근 : 하위 스코프에서 선언된 변수는 해당 스코프를 벗어나면 접근할 수 없습니다.

4.8 열거형 (Enum)

  • 열거형(enum)은 미리 정의된 상수 값들의 집합을 나타내는 특별한 데이터 타입입니다.
  • int, double, bool, String과 같은 기본 타입과 달리, enum은 사용자가 직접 관련된 상수 값들의 집합을 정의해야 합니다.
  • 예를 들어, ‘상태’를 나타낼 때 문자열(“pending”, “confirmed”) 대신 enum을 사용하면, 오타를 방지하고 코드의 의도를 더 명확하게 표현할 수 있습니다.
  • enum을 정의할 때는 enum 키워드와 이름을 사용하고, 중괄호({}) 안에 쉼표로 구분된 상수 값들을 나열합니다.

4.9 enum과 switch 문 활용

  • enum은 미리 정의된 상수 집합을 제공하므로, switch 문과 함께 사용하면 코드의 가독성과 안정성을 크게 높일 수 있습니다.
  • enum의 모든 가능한 상태를 switch 문의 case로 처리하면, 빠뜨리는 경우 없이 모든 상황에 대비할 수 있습니다.

5. Function

5.1 함수를 사용하는 이유

  • function, method, 함수는 모두 비슷한 의미로 사용되므로 이 문서에서는 구분하지 않습니다.
  • 함수가 프로그래밍에서 중요한 이유는 코드의 재사용성 과 유지보수성 을 높여주기 때문입니다.
  • 예를 들어, 여러 곳에서 동일한 기능이 필요할 때, 해당 기능을 함수로 만들어 두면 반복적인 코드 작성을 피할 수 있습니다.
  • 만약 기능에 수정이 필요할 경우, 함수 내부만 수정하면 해당 함수를 사용하는 모든 곳에 변경사항이 일괄적으로 적용됩니다.
  • 이는 코드의 일관성을 유지하고, 수정 과정에서 발생할 수 있는 버그를 줄여줍니다.
  • 이처럼 함수는 반복 작업을 줄이고, 코드를 더 체계적이고 효율적으로 관리하기 위해 반드시 필요합니다.

5.2 함수란 무엇인가

  • void main(){} –> 이것 자체가 function.

  • 입구 –> ()

  • 바디 –> {}

  • 출구 –> void

  • 출구 쪽으로 내보내는 방법 = return으로 값을 내보낸다.

  • 출구 값에 어떤 값이 나오는지 명시해줄 수 있다. int, double, string.

  • 어떤 데이터가 나올 것인지 미리 결정할 수 있다.

  • 또한 입구(변수)에 어떤 데이터 값을 넣어줄 것인지 명시해 줄 수 있다.

  • 입력값 String name

  • void main()은 입력값과 반환값이 없는 함수입니다. body {} 내부의 코드만 실행합니다. 따라서 return 할 것이 없으면 반환 타입을 void로 지정하거나 생략할 수 있습니다.

5.3 선택적 매개변수 (Optional Parameters)

  • 매개변수(parameter)는 함수에 전달하는 입력값을 말합니다.

  • 특정 매개변수를 선택적으로(optional) 만드려면, 매개변수 목록의 마지막에 대괄호([])로 감싸줍니다.

  • null은 ‘값이 없음’을 나타냅니다. 매개변수를 선택적으로 만들면 null 값이 전달될 수 있습니다.

  • 기본적으로 Dart의 변수는 null을 허용하지 않습니다. 예를 들어, int, double, String 타입의 변수에는 반드시 유효한 값이 할당되어야 합니다.

  • 변수가 null 값을 가질 수 있도록 하려면, 타입 뒤에 물음표(?)를 붙여 ‘nullable’ 타입으로 선언해야 합니다. (예: String?, int?)

  • Null 안정성(Null Safety)은 Dart의 중요한 특징입니다. 애플리케이션에서 발생하는 오류의 상당수는 null 값을 처리하지 않아 발생하기 때문입니다. ?를 사용하여 null 가능성을 명시적으로 다루면 코드의 안정성을 크게 높일 수 있습니다.

  • 선택적 매개변수는 여러 개를 선언할 수 있습니다.

5.4 Named Parameters

  • 선택적 매개변수가 많아지면, 각 값이 어떤 매개변수에 해당하는지 파악하기 어려울 수 있습니다.
  • 이 경우, 각 매개변수에 이름을 부여하여 key-value 형식으로 값을 전달하는 Named Parameters를 사용할 수 있습니다.
  • Named Parameters를 사용하려면 대괄호([]) 대신 중괄호({})로 매개변수 목록을 감쌉니다.
  • 중괄호({}) 안의 매개변수는 모두 선택적(optional)이며, 호출 시 매개변수 이름을 명시해야 합니다.

5.5 매개변수 기본값 (Default Parameter Values)

  • 함수를 호출할 때 특정 매개변수에 값을 전달하지 않아도 미리 정해진 기본값을 사용하도록 설정할 수 있습니다.
  • 이는 선택적 매개변수(Positional Optional Parameters 또는 Named Parameters)에만 적용할 수 있습니다.

1. 위치 기반 선택적 매개변수 (Positional Optional Parameters)에 기본값 설정

  • 대괄호([])로 감싸진 선택적 매개변수에 = 연산자를 사용하여 기본값을 할당합니다.

2. 명명된 매개변수 (Named Parameters)에 기본값 설정

  • 중괄호({})로 감싸진 명명된 매개변수에도 = 연산자를 사용하여 기본값을 할당할 수 있습니다.
  • 매개변수에 기본값이 설정되어 있으면, 해당 매개변수는 null이 될 수 없으므로 String?처럼 nullable 타입으로 선언할 필요가 없습니다.

5.6 좋은 함수 작성법: 순수 함수 (Pure Functions)

  • 좋은 함수를 작성하는 중요한 원칙 중 하나는 순수 함수(Pure Function) 처럼 만드는 것입니다.
  • 순수 함수는 다음과 같은 특징을 가집니다.
  1. 동일한 입력에 대해 항상 동일한 출력: 함수에 같은 값을 여러 번 전달하면 항상 같은 결과가 반환됩니다.
  2. 부작용(Side Effect) 없음: 함수 외부의 어떤 상태(전역 변수, 데이터베이스, 화면 등)도 변경하지 않습니다. 또한, 함수 외부의 상태에 의존하지도 않습니다.

예시:

  • 순수 함수는 코드를 예측하기 쉽게 만들고, 테스트하기 용이하며, 병렬 처리에도 안전하다는 장점이 있습니다.

5.7 좋은 함수 작성법: 단일 책임과 명확한 이름

  • 좋은 함수를 작성하는 또 다른 중요한 원칙은 단일 책임 원칙(Single Responsibility Principle) 을 따르고, 함수의 이름을 명확하게 짓는 것입니다.

1. 단일 책임 원칙 (Single Responsibility Principle)

  • 함수는 오직 한 가지 작업만 수행해야 합니다.
  • 한 함수가 여러 가지 일을 처리하려고 하면, 코드가 복잡해지고 유지보수가 어려워집니다.

2. 명확하고 서술적인 이름

  • 함수의 이름은 함수가 무엇을 하는지 명확하게 설명해야 합니다.
  • 짧고 모호한 이름보다는 길더라도 함수의 목적을 정확히 나타내는 이름이 좋습니다.

5.8 익명 함수와 콜백 (Anonymous Functions and Callbacks)

  • Dart에서 함수는 일급 객체(first-class object)입니다. 이는 함수를 변수에 할당하거나, 다른 함수의 매개변수로 전달하거나, 함수의 반환값으로 사용할 수 있다는 의미입니다.

1. 익명 함수 (Anonymous Functions)

  • 이름이 없는 함수를 익명 함수(Anonymous Function) 라고 부릅니다.
  • 익명 함수는 주로 다른 함수의 매개변수로 전달되거나, 변수에 할당되어 필요할 때 호출됩니다.

2. 콜백 함수 (Callback Functions)

  • 다른 함수의 매개변수로 전달되어, 특정 이벤트가 발생하거나 작업이 완료되었을 때 호출되는 함수를 콜백 함수(Callback Function) 라고 합니다.
  • 익명 함수는 콜백 함수로 자주 사용됩니다.

6. Classes

6.1 클래스란 무엇인가

  • 클래스는 객체가 가질 데이터(속성, properties)와 객체가 수행할 동작(메서드, methods)을 정의합니다.
  • 클래스(Class): ‘붕어빵 틀’이나 ‘자동차 설계도’에 해당합니다.
  • 인스턴스(Instance) 또는 객체(Object): 클래스라는 틀을 사용하여 실제로 만들어진 ‘붕어빵’이나 ‘자동차’ 개체를 의미합니다.

6.2 커스텀 데이터 타입 (Custom Data Types)

  • Dart는 int(정수), String(문자열) 등 기본적인 데이터 타입을 제공합니다. 하지만 복잡한 데이터를 표현하기에는 한계가 있습니다.
  • 예를 들어, ‘사용자’를 표현하려면 이름(String), 나이(int), 로그인 여부(bool) 등 여러 데이터가 하나의 단위로 묶여야 합니다.
  • 이처럼 여러 데이터를 묶어 우리만의 새로운 데이터 타입을 만드는 것을 ‘커스텀 데이터 타입’이라고 하며, 이때 class를 사용합니다.
  • class를 사용하면 관련된 데이터(속성)와 기능(메서드)을 하나의 ‘사용자’ 타입으로 캡슐화하여, 코드를 더 명확하고 구조적으로 만들 수 있습니다.
  • 흥미롭게도 Dart에서는 int와 같은 기본 타입조차 내부적으로는 클래스로 구현된 객체입니다. 이는 Dart가 일관성 있는 객체 지향 언어임을 보여줍니다.

6.3 생성자 (Constructor)

  • 생성자(Constructor)는 클래스의 인스턴스(객체)를 생성할 때 호출되는 특별한 함수입니다.
  • 생성자의 주된 역할은 인스턴스가 처음 만들어질 때 필요한 초기화 작업을 수행하는 것입니다. 예를 들어, 클래스의 속성(변수)에 초기값을 할당할 수 있습니다.
  • 생성자는 클래스와 이름이 같아야 하며, 반환 타입을 지정하지 않습니다.

  • 하지만 Dart 2부터 new 키워드는 선택사항(optional)이 되었으며, 생략하는 것이 권장되는 스타일입니다.

6.5 인스턴스 속성 접근 및 변경

  • 클래스로부터 인스턴스를 생성한 후, 점(.) 연산자를 사용하여 해당 인스턴스의 속성(변수)에 접근하거나 값을 변경할 수 있습니다.
  • 이처럼 인스턴스의 속성 값은 생성된 이후에도 자유롭게 변경할 수 있습니다. (단, final로 선언된 속성은 제외)

  • 인스턴스를 출력했을 때 우리가 원하는 유용한 정보를 표시하려면, Object 클래스로부터 상속받은 toString 메서드를 재정의(Override) 해야 합니다.
  • toString 메서드는 해당 객체를 나타내는 문자열(String)을 반환하도록 직접 구현할 수 있습니다.
  • @override 어노테이션은 부모 클래스의 메서드를 재정의한다는 것을 명시적으로 나타내며, 실수를 방지하는 데 도움을 줍니다.

6.7 클래스 내부 함수 (메서드)

  • 클래스 내부에 정의된 함수를 메서드(Method) 라고 부릅니다.
  • 메서드는 클래스가 가진 속성(데이터)을 사용하거나 변경하는 등, 해당 클래스와 관련된 동작을 수행합니다.
  • 예를 들어, Tesla 클래스에 ‘주행한다’ 또는 ‘충전한다’와 같은 동작을 메서드로 추가할 수 있습니다.
  • 인스턴스의 메서드를 호출할 때도 속성과 마찬가지로 점(.) 연산자를 사용합니다.

6.9 명명된 생성자 (Named Constructors)

  • 하나의 클래스에는 여러 개의 생성자를 가질 수 있습니다. 이를 통해 다양한 방법으로 객체를 생성할 수 있습니다.
  • 기본 생성자 외에 추가적인 생성자를 만들려면 명명된 생성자(Named Constructor) 를 사용합니다.
  • 명명된 생성자는 ClassName.constructorName() 형식으로 이름을 지어 만듭니다.

  • 이처럼 명명된 생성자는 특정 목적이나 값을 가진 객체를 더 명확하고 편리하게 생성하는 방법을 제공합니다.

6.10 생성자와 속성 초기화

  • Dart의 Null Safety 규칙에 따라, non-nullable(타입 뒤에 ?가 없는) 속성은 선언과 동시에 또는 생성자를 통해 반드시 초기화되어야 합니다.
  • 매개변수에 기본값 제공하기:
    • 함수에서와 마찬가지로, 생성자의 매개변수에도 기본값을 지정할 수 있습니다.
    • 이는 선택적 매개변수(optional parameters)와 함께 사용됩니다.

6.11 필수 명명된 매개변수 (Required Named Parameters)

  • 5.4에서 배운 명명된 매개변수({})는 기본적으로 선택적(optional)입니다.
  • 하지만 명명된 매개변수를 사용하면서도 특정 값을 반드시 전달하도록 강제하고 싶을 때가 있습니다. 이때 required 키워드를 사용합니다.
  • required는 해당 매개변수가 null이 될 수 없으며, 객체 생성 시 반드시 값을 전달해야 함을 의미합니다.
  • required 키워드는 Null Safety와 함께 코드의 안정성을 높이는 매우 중요한 기능입니다.

6.12 명명된 매개변수 기본값

  • 명명된 매개변수({})는 required로 표시하지 않으면 선택적(optional)이며, 값을 전달하지 않으면 기본적으로 null이 됩니다.
  • 매개변수가 null이 되는 것을 원치 않거나, 특정 기본값을 가지게 하고 싶을 때 대입 연산자(=)를 사용하여 기본값을 지정할 수 있습니다.
  • 기본값이 있는 매개변수는 non-nullable(String)로 선언할 수 있습니다. 왜냐하면 값이 전달되지 않아도 null이 아닌 기본값이 할당되는 것이 보장되기 때문입니다.

6.13 파일 분리 및 가져오기 (import)

  • 프로젝트가 커지면, 관련된 클래스나 함수들을 별도의 파일로 분리하여 관리하는 것이 좋습니다. 이는 코드의 재사용성과 유지보수성을 크게 향상시킵니다.

예시:

  1. tesla.dart 파일 생성:
    먼저, Tesla 클래스를 별도의 tesla.dart 파일로 분리하여 저장합니다.

  2. main.dart 파일에서 가져오기:
    이제 다른 파일(예: main.dart)에서 import 키워드를 사용하여 tesla.dart 파일의 내용을 가져와 Tesla 클래스를 사용할 수 있습니다.

  • import 문에는 현재 파일을 기준으로 한 상대 경로를 사용합니다. 만약 lib 디렉토리 안에 있다면 package: 접두사를 사용할 수도 있습니다.

6.14 Private 변수와 정보 은닉

  • 객체 지향 프로그래밍에서는 클래스 내부의 중요한 데이터나 상태를 외부로부터 보호하는 정보 은닉(Information Hiding) 이 중요합니다.

  • Dart에서는 다른 언어의 private 키워드 대신, 변수나 메서드 이름 앞에 언더스코어(_)를 붙여 private으로 만듭니다.

  • 중요: Dart의 private은 라이브러리-프라이빗(library-private) 입니다. 이는 클래스-프라이빗(class-private)과 달리, 같은 파일 내에서는 접근할 수 있고 다른 파일에서는 접근할 수 없음을 의미합니다.

예시:

  1. user.dart 파일:
    _password라는 private 속성을 가진 User 클래스를 정의합니다.

  2. main.dart 파일:
    다른 파일에서 _password에 접근을 시도합니다.

  • 이처럼 _를 사용하여 외부로부터의 직접적인 접근을 막고, 클래스가 의도한 방식으로만 데이터가 사용되도록 강제하여 코드의 안정성을 높일 수 있습니다.

6.15 초기화 리스트 (Initializer Lists)

  • 생성자 바디({})가 실행되기 전에 인스턴스 변수를 초기화해야 할 때 초기화 리스트(Initializer List) 를 사용합니다.

  • 생성자 매개변수 목록 뒤에 콜론(:)을 붙여 사용하며, 쉼표(,)로 여러 초기화 표현식을 연결할 수 있습니다.

  • final 변수를 초기화하는 유일한 방법이기도 합니다.

  • 주요 사용 사례:

    1. final 필드 초기화
    2. assert를 사용하여 입력값 검증
    3. 부모 클래스의 생성자 호출 (super)

예시:

6.16 상수 생성자 (Const Constructors)

  • 클래스가 완전히 불변(immutable) 하고, 모든 속성이 final이며, 생성자의 인자들로만 초기화될 때 상수 생성자(Const Constructor) 를 만들 수 있습니다.
  • 생성자 앞에 const 키워드를 붙여서 정의합니다.

상수 생성자의 조건:

  1. 클래스의 모든 인스턴스 변수는 반드시 final이어야 합니다.
  2. 생성자는 this. 문법을 사용하거나 초기화 리스트를 통해 final 변수들을 초기화해야 합니다.

상수 생성자의 장점: 성능 향상

  • const 생성자로 동일한 인자를 사용하여 여러 번 인스턴스를 생성하면, Dart는 이들을 동일한 하나의 인스턴스(canonical instance) 로 취급합니다.
  • 이는 메모리 사용량을 줄이고 앱의 성능을 향상시키는 중요한 최적화 기법입니다. 위젯 트리(Widget Tree)를 사용하는 Flutter에서 특히 중요합니다.

7. Nullability

7.1 Null 개요 (Introduction to Null)

  • null 은 ‘값이 없음’을 나타내는 특별한 값입니다. 프로그래밍에서 변수가 어떤 유효한 데이터도 가지고 있지 않을 때 null을 사용합니다.

null의 필요성:

  • 예를 들어, ‘전화번호’를 저장하는 변수가 있다고 가정해 봅시다. 모든 사람이 전화번호를 가지고 있는 것은 아닙니다.
  • 이 경우, 전화번호가 없는 상태를 null로 표현하는 것이 가장 명확합니다. ‘없음’이나 ‘none’과 같은 문자열로 표현하면, 실제 데이터와 혼동될 수 있습니다.

Null Safety (Dart 2.12부터)

  • Dart는 버전 2.12부터 Null Safety 기능을 도입했습니다. 이는 개발자가 null 값을 안전하게 다루도록 돕는 강력한 기능입니다.
  • 기본 원칙: Dart의 모든 변수는 기본적으로 null을 허용하지 않습니다. 즉, String, int와 같이 선언된 변수에는 null을 할당할 수 없습니다.
  • 도입 배경: 프로그램에서 발생하는 오류의 상당수가 null 값을 예상치 못하게 사용하여 발생합니다. Null Safety는 이러한 오류를 컴파일 시점에 미리 방지하여 코드의 안정성을 크게 높여줍니다.

Null Safety 예시:

  • 이처럼 Null Safety는 null로 인한 잠재적인 런타임 오류를 컴파일 단계에서 잡아내어, 더 견고하고 예측 가능한 코드를 작성하도록 돕습니다.

7.2 Nullable 타입 (Nullable Types)

  • Dart의 Null Safety는 변수가 기본적으로 null을 허용하지 않는다고 배웠습니다.
  • 하지만 때로는 변수가 null 값을 가질 수 있도록 허용해야 할 필요가 있습니다. 예를 들어, 사용자 프로필에서 ‘선택 사항’인 전화번호 필드 등이 그렇습니다.
  • 변수가 null을 가질 수 있도록 허용하려면, 타입 뒤에 물음표(?)를 붙여 Nullable 타입으로 선언합니다.

예제 코드:

  • String?와 String은 서로 다른 타입입니다. String?는 null 또는 String 값을 가질 수 있지만, String은 String 값만 가질 수 있습니다.

7.3 Nullable 타입 안전하게 다루기

  • Nullable 타입의 변수는 null일 수도 있고 아닐 수도 있습니다. 따라서 null이 아닌 경우에만 접근할 수 있는 속성이나 메서드를 사용하려면, 해당 변수가 null이 아님을 Dart 컴파일러에게 알려주어야 합니다.
  • null 가능성이 있는 변수의 멤버에 직접 접근하려고 하면 컴파일 에러가 발생합니다.

문제 상황:

해결 방법:

1. null이 아님을 명시적으로 확인 (Null Check)

  • if 문을 사용하여 변수가 null이 아님을 명시적으로 확인하면, Dart 컴파일러는 해당 블록 내에서 변수가 non-nullable임을 보장합니다.

2. 값 할당 (Initialization)

  • 변수에 null이 아닌 값을 할당하면, 그 시점부터 변수는 non-nullable처럼 동작합니다.

7.4 Null-aware 연산자 (Null-aware Operators)

  • Dart는 null 값을 안전하고 간결하게 다룰 수 있도록 다양한 Null-aware 연산자를 제공합니다.

1. ?? (If-null 연산자)

  • 표현식이 null이면 ?? 뒤의 값을 사용하고, null이 아니면 표현식의 값을 사용합니다.

2. ??= (Null-aware 할당 연산자)

  • 변수의 값이 null일 때만 새로운 값을 할당합니다. null이 아니면 기존 값을 유지합니다.

3. ?. (Null-aware 접근 연산자)

  • null 가능성이 있는 객체의 멤버(속성 또는 메서드)에 안전하게 접근합니다.
  • 객체가 null이 아니면 멤버에 접근하고, null이면 null을 반환합니다. 에러를 발생시키지 않습니다.

4. ! (Null Assertion 연산자)

  • 개발자가 해당 변수가 절대로 null이 아님을 확신할 때 사용합니다.
  • !를 사용하면 nullable 변수를 non-nullable처럼 사용할 수 있습니다.
  • 주의: 만약 변수가 실제로 null인데 !를 사용하면, 런타임 에러(Null check operator used on a null value)가 발생합니다.

5. ?.. (Null-aware Cascade 연산자)

  • .. (Cascade 연산자)와 유사하지만, 객체가 null이 아닐 때만 연산을 수행합니다.
  • 객체가 null이면 아무 작업도 수행하지 않고 null을 반환합니다.

6. ?[] (Null-aware 인덱스 연산자)

  • null 가능성이 있는 리스트나 맵에 인덱스로 접근할 때 사용합니다.
  • 컬렉션이 null이 아니면 인덱스에 접근하고, null이면 null을 반환합니다.

7.5 Non-Nullable 변수 초기화 (Non-Nullable Variable Initialization)

  • Dart의 Null Safety 규칙에 따라, non-nullable 변수(타입 뒤에 ?가 없는 변수)는 선언과 동시에 또는 생성자를 통해 반드시 null이 아닌 값으로 초기화되어야 합니다.
  • 초기화되지 않은 non-nullable 변수는 컴파일 에러를 발생시킵니다.

non-nullable 변수를 초기화하는 주요 방법:

  1. 선언 시 초기값 할당: 가장 기본적인 방법입니다.
  2. 생성자 매개변수 사용: 생성자를 통해 값을 받아 초기화합니다.
    • this.를 사용한 단축 문법
    • 초기화 리스트(:) 사용
    • required 키워드를 사용한 명명된 매개변수
  3. 기본값 제공: 선택적 매개변수에 기본값을 할당합니다.

7.6 Nullable 변수와 흐름 분석 (Flow Analysis)

  • Dart의 Null Safety는 컴파일러가 코드의 흐름을 분석하여 변수가 null이 될 수 있는지 없는지를 판단하는 흐름 분석(Flow Analysis) 기능을 제공합니다.
  • 이 기능 덕분에 if (변수 != null)과 같은 null 체크 후에는 해당 변수를 non-nullable처럼 안전하게 사용할 수 있습니다.
  • 하지만 이 흐름 분석은 지역 변수(Local Variable) 와 인스턴스 변수(Instance Variable) 에 대해 다르게 작동합니다.

1. 지역 변수 (Local Variable)의 흐름 분석

  • 지역 변수는 해당 스코프 내에서만 유효하며, 다른 코드에 의해 갑자기 변경될 가능성이 적습니다.
  • 따라서 null 체크 후에는 컴파일러가 해당 지역 변수가 null이 아님을 확신하고 non-nullable처럼 사용하도록 허용합니다.

2. 인스턴스 변수 (Instance Variable)의 흐름 분석

  • 인스턴스 변수는 클래스의 메서드 외부에서도 변경될 수 있습니다.
  • 따라서 null 체크를 했더라도, 컴파일러는 해당 인스턴스 변수가 null 체크 이후에 다른 코드에 의해 null로 변경될 가능성을 배제할 수 없습니다. 이로 인해 null 체크 후에도 멤버 접근 시 에러가 발생할 수 있습니다.

8. Collections

8.1 컬렉션 소개 (Introduction to Collections)

  • 컬렉션(Collection) 은 여러 개의 데이터를 효율적으로 저장하고 관리하기 위한 자료구조입니다. Dart는 다음과 같은 주요 컬렉션 타입을 제공합니다.

  • Lists: 데이터를 순서대로 저장합니다. (다른 언어의 배열과 유사)

  • Sets: 데이터를 나열하되, 중복을 허용하지 않습니다.

  • Maps: Key-Value 쌍으로 데이터를 저장합니다.

8.2 리스트 (Lists)

  • 리스트(List) 는 여러 데이터를 순서대로 저장하는 컬렉션입니다. 다른 프로그래밍 언어의 ‘배열(Array)’과 유사하며, 각 요소는 0부터 시작하는 인덱스(index)를 통해 접근할 수 있습니다.

1. 리스트 생성

  • 리스트를 생성할 때는 대괄호([])를 사용합니다. List<타입>과 같이 타입을 명시하거나, var를 사용하여 Dart가 타입을 추론하도록 할 수 있습니다.

2. 리스트 요소 접근 및 수정

  • 인덱스를 사용하여 리스트의 특정 요소에 접근하거나 값을 변경할 수 있습니다.

3. 리스트 요소 추가 및 삭제

  • add(), remove() 메서드를 사용하여 리스트에 요소를 추가하거나 삭제할 수 있습니다.

8.3 리스트의 불변성 (List Immutability)

  • 리스트는 기본적으로 가변(mutable) 컬렉션입니다. 즉, 생성된 후에 요소를 추가, 삭제, 수정할 수 있습니다.
  • 하지만 때로는 리스트의 내용이 변경되지 않도록 불변(immutable) 으로 만들 필요가 있습니다. Dart에서는 final과 const 키워드를 사용하여 리스트의 불변성을 제어할 수 있습니다.

1. final 리스트

  • final 키워드는 리스트 변수 자체의 재할당을 막습니다. 즉, final로 선언된 리스트 변수에는 다른 리스트를 다시 할당할 수 없습니다.
  • 하지만 리스트 내부의 요소는 여전히 변경 가능합니다.

2. const 리스트

  • const 키워드는 리스트 변수 자체의 재할당을 막을 뿐만 아니라, 리스트 내부의 모든 요소도 불변으로 만듭니다.
  • const 리스트는 컴파일 시점에 값이 결정되어야 합니다.

3. 런타임에 결정되는 불변 리스트

  • 리스트의 내용이 런타임에 결정되지만, 생성된 후에는 변경되지 않도록 불변으로 만들고 싶을 때 List.unmodifiable() 팩토리 생성자를 사용합니다.
  • 핵심: final은 ‘변수’를 불변으로 만들고, const는 ‘변수와 그 내용’을 모두 불변으로 만들며, List.unmodifiable()는 ‘내용’을 불변으로 만듭니다.

8.15 고차 메서드: where(), reduce(), fold()

  • map()과 마찬가지로 where(), reduce(), fold()는 컬렉션을 다루는 데 매우 유용한 고차(Higher-Order) 메서드입니다.

1. where()로 필터링

  • where() 메서드는 컬렉션의 요소 중 특정 조건을 만족하는 요소들만 선택하여 새로운 Iterable을 반환합니다.

2. reduce()로 컬렉션 통합

  • reduce() 메서드는 컬렉션의 모든 요소를 하나의 단일 값으로 줄일 때 사용합니다.
  • 컬렉션이 비어있으면 에러가 발생하므로, 컬렉션이 비어있지 않음이 확실할 때 사용해야 합니다.

3. fold()로 안전하게 통합

  • fold() 메서드는 reduce()와 유사하게 컬렉션의 모든 요소를 하나의 값으로 줄이지만, 초기값(initial value) 을 제공한다는 점에서 차이가 있습니다.
  • 초기값을 제공하므로 컬렉션이 비어있어도 에러가 발생하지 않고, 초기값을 반환합니다.

8.16 고차 메서드: sort(), reversed

  • 컬렉션의 요소를 정렬하거나 순서를 뒤집을 때 sort() 메서드와 reversed 속성을 사용합니다.

1. sort() 메서드

  • sort() 메서드는 리스트의 요소를 오름차순으로 정렬합니다.
  • 주의: sort()는 원본 리스트 자체를 변경(mutable)합니다.

2. reversed 속성

  • reversed 속성은 리스트의 요소를 역순으로 반환하는 새로운 Iterable을 생성합니다.

    • 음수: 첫 번째 요소가 두 번째 요소보다 작음

8.17 컬렉션 선택 가이드 (Collection Selection Guide)

  • Dart에서 제공하는 List, Set, Map은 각각 고유한 특징을 가지고 있으므로, 프로그램의 요구사항에 맞춰 적절한 컬렉션 타입을 선택하는 것이 중요합니다.

1. 리스트 (List)

  • 주요 특징: 순서가 있으며, 중복된 요소를 허용합니다.
  • 사용 시점:
    • 요소의 순서가 중요할 때 (예: 사용자 인터페이스의 항목 순서)
    • 동일한 요소가 여러 번 나타날 수 있을 때
    • 리스트의 끝에 요소를 자주 추가하거나 삭제할 때
  • 성능 고려: 요소가 많아지면 특정 요소의 검색(탐색) 성능이 선형적으로 감소할 수 있습니다 (O(n)).

2. 세트 (Set)

  • 주요 특징: 순서가 없으며, 모든 요소가 고유해야 합니다 (중복 불허용).
  • 사용 시점:
    • 요소의 중복을 허용하지 않고 고유한 값들만 관리할 때 (예: 태그 목록, 방문자 ID)
    • 특정 요소가 컬렉션에 포함되어 있는지 빠르게 확인해야 할 때 (O(1)에 가까운 검색 성능)
  • 성능 고려: 요소의 포함 여부 확인이 매우 빠릅니다.

3. 맵 (Map)

  • 주요 특징: 키(Key)와 값(Value)의 쌍으로 데이터를 저장하며, 키는 고유해야 합니다.
  • 사용 시점:
    • 특정 키를 사용하여 값에 빠르게 접근해야 할 때 (예: 사용자 ID로 사용자 정보 조회)
    • 데이터가 키-값 쌍의 형태로 구성될 때 (예: JSON 데이터, 설정 정보)
  • 성능 고려: 키를 통한 값 조회 성능이 매우 빠릅니다 (O(1)에 가까움).

9.1 클래스 상속 (Class Inheritance)

  • 상속(Inheritance) 은 객체 지향 프로그래밍의 핵심 개념 중 하나로, 기존 클래스(부모 클래스 또는 슈퍼클래스)의 속성과 메서드를 새로운 클래스(자식 클래스 또는 서브클래스)가 물려받아 재사용하고 확장할 수 있도록 하는 기능입니다.

상속의 장점:

  • 코드 재사용: 부모 클래스에 정의된 코드를 자식 클래스에서 다시 작성할 필요가 없습니다.

  • 유지보수성: 공통된 로직을 부모 클래스에 모아 관리할 수 있습니다.

9.2 메서드 재정의 (Method Override)

  • 자식 클래스가 부모 클래스로부터 상속받은 메서드의 동작을 자신에게 맞게 변경해야 할 때 메서드 재정의(Method Override) 를 사용합니다.

  • @override 어노테이션을 사용하여 해당 메서드가 부모 클래스의 메서드를 재정의한다는 것을 명시적으로 나타내는 것이 좋습니다. 이는 코드의 가독성을 높이고, 오타 등으로 인한 실수를 컴파일러가 잡아주도록 돕습니다.

  • super 키워드는 자식 클래스에서 부모 클래스의 멤버(속성 또는 메서드) 에 접근할 때 사용됩니다.
  • 주로 부모 클래스의 메서드를 재정의하면서도, 부모의 원래 동작을 유지하거나 확장하고 싶을 때 유용하게 사용됩니다.

9.4 다단계 상속 (Multilevel Inheritance)

  • 다단계 상속은 한 클래스가 다른 클래스를 상속하고, 그 클래스가 또 다른 클래스를 상속하는 계층 구조를 의미합니다.
  • 이는 현실 세계의 계층적 분류를 코드에 반영할 때 유용합니다.

상속 관계의 이해:

  • 부모-자식 관계: 상위 클래스(부모)와 하위 클래스(자식) 간의 관계입니다. 자식 클래스는 부모 클래스의 모든 멤버를 물려받습니다.
  • 형제 관계: 같은 부모 클래스를 상속받는 여러 자식 클래스 간의 관계입니다. 형제 클래스들은 서로 직접적인 상속 관계가 없습니다.

예제 코드:

  • 핵심: 자식 클래스는 부모 클래스의 모든 멤버를 물려받으며, 다단계 상속에서는 최하위 자식 클래스가 모든 상위 부모 클래스의 멤버를 물려받습니다.

9.5 상속 사용 시 고려사항 (Considerations for Inheritance)

  • 상속은 강력한 기능이지만, 잘못 사용하면 코드의 복잡성을 증가시키고 유지보수를 어렵게 만들 수 있습니다. 따라서 상속을 사용할 때는 다음과 같은 사항들을 고려하는 것이 중요합니다.

  • 너무 깊은 상속 계층 피하기:

    • 클래스 계층이 너무 깊어지면(예: 4단계 이상) 코드의 이해와 추적이 어려워집니다. 일반적으로 2~3단계의 상속 계층을 유지하는 것이 좋습니다.

  • 컴포지션(Composition) 우선 고려:

    • 상속은 ‘is-a’ 관계(예: Dog는 Animal이다)일 때 적합합니다.
    • ‘has-a’ 관계(예: Car는 Engine을 가진다)일 때는 상속 대신 컴포지션(Composition), 즉 다른 클래스의 인스턴스를 속성으로 포함하는 방식을 먼저 고려하는 것이 좋습니다. 컴포지션은 더 유연한 코드 설계를 가능하게 합니다.

  • ‘is-a’ 관계에만 상속 사용:

    • 상속은 자식 클래스가 부모 클래스의 한 종류임을 나타낼 때만 사용해야 합니다. 이는 코드의 의미론적 정확성을 유지하는 데 중요합니다.

9.7 추상 클래스 (Abstract Classes)

  • 추상 클래스(Abstract Class) 는 다른 클래스들이 상속받아 구현해야 할 공통된 특징이나 동작을 정의하지만, 그 자체로는 완전한 기능을 하지 못하는 클래스입니다.
  • 추상 클래스는 직접 인스턴스(객체)를 생성할 수 없으며, 반드시 다른 클래스에 의해 상속(extends)되어야 합니다.

추상 클래스의 특징:

  1. abstract 키워드 사용: 클래스 선언 앞에 abstract 키워드를 붙입니다.
  2. 인스턴스 생성 불가: new AbstractClass()와 같이 직접 객체를 만들 수 없습니다.
  3. 추상 메서드 포함 가능: 바디({})가 없는 메서드(추상 메서드)를 가질 수 있습니다. 추상 메서드는 자식 클래스에서 반드시 구현(override)해야 합니다.
  4. 일반 메서드 및 속성 포함 가능: 추상 클래스도 일반 클래스처럼 구현된 메서드나 속성을 가질 수 있습니다.

9.9 extends vs implements (상속 vs 구현)

  • Dart에서 클래스 간의 관계를 설정하는 두 가지 주요 방법은 extends (상속)와 implements (구현)입니다. 이 둘은 비슷해 보이지만 중요한 차이가 있습니다.

1. extends (상속)

  • extends는 부모 클래스의 구현(코드) 을 물려받습니다.
  • 부모 클래스의 속성과 메서드를 자식 클래스에서 그대로 사용하거나, 필요에 따라 재정의(@override)할 수 있습니다.
  • ‘is-a’ 관계 (예: Dog는 Animal이다)에 사용됩니다.
  • 단일 상속만 가능합니다. (하나의 클래스만 extends할 수 있습니다.)

2. implements (구현)

  • implements는 인터페이스(클래스의 ‘청사진’ 또는 ‘계약’)를 구현합니다.
  • 인터페이스에 정의된 모든 속성과 메서드를 자식 클래스에서 반드시 직접 구현해야 합니다. (부모의 구현을 물려받지 않습니다.)
  • ‘can-do’ 관계 (예: Car는 Drive할 수 있다)에 사용됩니다.
  • 다중 구현이 가능합니다. (여러 인터페이스를 implements할 수 있습니다.)

9.11 믹스인 (Mixins)

  • 믹스인(Mixin) 은 클래스에 특정 기능을 추가하는 유연한 방법입니다. 상속(extends)은 단일 상속만 가능하고, 인터페이스(implements)는 구현을 강제하지만 코드를 재사용할 수 없다는 한계가 있습니다.
  • 믹스인은 이러한 한계를 극복하고, 여러 클래스에 걸쳐 코드를 재사용할 수 있도록 해줍니다.

믹스인이 필요한 문제 상황:

  • 예를 들어, Bird 클래스가 fly()와 layEggs() 메서드를 가진다고 가정해 봅시다.
  • Pigeon은 fly()도 하고 layEggs()도 합니다.
  • Ostrich는 layEggs()는 하지만 fly()는 하지 못합니다.
  • Fish는 fly()도 layEggs()도 하지 않습니다.
  • 이처럼 모든 Bird가 fly()하는 것은 아니며, layEggs()하는 것이 Bird만의 특징도 아닙니다. (예: Turtle도 layEggs()를 합니다.)
  • 이럴 때 fly()와 layEggs() 기능을 각각의 믹스인으로 분리하여 필요한 클래스에만 추가할 수 있습니다.

믹스인 정의 및 사용:

  • 믹스인은 mixin 키워드를 사용하여 정의합니다.
  • 클래스에 믹스인을 추가할 때는 with 키워드를 사용합니다.

10. Asynchronous Programming

10.1 비동기 프로그래밍 소개 (Introduction to Asynchronous Programming)

  • Dart는 기본적으로 단일 스레드(Single-threaded) 언어입니다. 이는 한 번에 하나의 작업만 순차적으로 처리한다는 의미입니다.
  • 만약 메인 스레드에서 네트워크 요청이나 파일 읽기처럼 시간이 오래 걸리는 작업을 직접 처리하게 되면, 그 작업이 완료될 때까지 다른 모든 작업(예: 사용자 인터페이스 업데이트)이 멈추게 됩니다. 이를 UI 멈춤(UI Freezing) 또는 블로킹(Blocking) 이라고 합니다.

비동기 프로그래밍의 필요성:

  • 특히 사용자 인터페이스가 중요한 애플리케이션(예: Flutter 앱)에서는 UI가 부드럽게 동작하는 것이 중요합니다. 일반적으로 60fps(초당 60프레임)를 유지하려면 각 프레임을 16ms(밀리초) 안에 처리해야 합니다.
  • 오래 걸리는 작업을 메인 스레드에서 직접 처리하면 이 16ms를 초과하여 UI가 버벅거리거나 멈추게 됩니다.
  • 해결책: 오래 걸리는 작업을 메인 스레드에서 직접 처리하는 대신, 비동기적으로(Asynchronously) 처리하여 메인 스레드를 블로킹하지 않도록 합니다. 비동기 작업은 백그라운드에서 진행되고, 완료되면 그 결과를 메인 스레드로 다시 전달합니다.

예제 코드 (블로킹 문제):

  • 위 예제처럼 오래 걸리는 작업을 동기적으로 실행하면, ‘메인 스레드 시작’ 메시지 이후 ‘오래 걸리는 작업 완료’ 메시지가 출력될 때까지 상당한 지연이 발생합니다. 이 시간 동안 UI는 멈추게 됩니다.
  • Dart의 비동기 프로그래밍은 이러한 문제를 Future와 Stream을 통해 해결합니다.

10.2 Future 소개 (Introduction to Future)

  • Future 는 Dart에서 비동기 작업을 다루는 핵심 객체입니다. ‘미래에 완료될 작업’ 또는 ‘미래에 얻게 될 값’을 나타냅니다.
  • Future는 비동기 작업의 결과를 즉시 반환하는 대신, 작업이 완료되면 그 결과를 알려주겠다는 ‘약속’과 같습니다.

Future의 세 가지 상태:

  1. Uncompleted (미완료): 비동기 작업이 아직 진행 중이며, 결과가 아직 결정되지 않은 상태입니다.
  2. Completed with error (오류 완료): 비동기 작업이 완료되었지만, 도중에 오류가 발생하여 실패한 상태입니다.
  3. Completed with data (데이터 완료): 비동기 작업이 성공적으로 완료되어, 예상했던 결과 데이터(값)를 가진 상태입니다.

10.3 Future 처리하기: then(), catchError()

  • Future 객체는 비동기 작업의 결과를 ‘미래’에 알려주므로, 그 결과가 준비되었을 때 특정 동작을 수행하도록 코드를 작성해야 합니다.
  • Future의 결과를 처리하는 가장 기본적인 방법은 then()과 catchError() 메서드를 사용하는 것입니다.

예제 코드:

  • then(): Future가 성공적으로 완료되어 데이터가 준비되었을 때 호출되는 콜백 함수를 등록합니다.
  • catchError(): Future가 오류와 함께 완료되었을 때 호출되는 콜백 함수를 등록합니다.
  • whenComplete(): Future의 성공 또는 실패 여부와 관계없이, Future가 완료되면 항상 호출되는 콜백 함수를 등록합니다.

10.4 Future.delayed()로 비동기 작업 만들기

  • Future.delayed()는 특정 시간(Duration)이 지난 후에 완료되는 Future를 생성할 때 사용합니다.
  • 이는 네트워크 요청이나 파일 읽기처럼 실제 시간이 소요되는 비동기 작업을 시뮬레이션하거나, 특정 시간 지연이 필요한 경우에 유용합니다.

예제 코드:

  • Future.delayed()는 비동기 작업의 기본 개념을 이해하는 데 좋은 예시가 됩니다.

10.5 async와 await (Async and Await)

  • async와 await 키워드는 Future를 사용하여 비동기 코드를 작성할 때, 마치 동기 코드처럼 순차적으로 읽히고 이해하기 쉽게 만들어주는 문법적 설탕(syntactic sugar)입니다.
  • then()과 catchError()를 사용하는 것보다 코드를 더 간결하고 직관적으로 만들 수 있습니다.

사용법:

  1. async: await 키워드를 사용하려는 함수 앞에 async를 붙여, 해당 함수가 비동기 작업을 포함하고 Future를 반환할 수 있음을 나타냅니다.
  2. await: Future를 반환하는 비동기 작업 앞에 await를 붙여, 해당 Future가 완료될 때까지 기다렸다가 그 결과를 받아옵니다. await는 async 함수 내에서만 사용할 수 있습니다.

예제 코드:

  • async 함수는 항상 Future를 반환합니다. 만약 Future가 아닌 값을 return하면, Dart는 자동으로 그 값을 Future.value()로 감싸서 반환합니다.

10.6 비동기 에러 처리: try-catch와 Future 메서드

  • 비동기 작업 중에는 네트워크 문제, 서버 오류, 데이터 파싱 실패 등 다양한 예외(Exception)가 발생할 수 있습니다.
  • Dart에서는 이러한 비동기 예외를 안전하게 처리하기 위해 try-catch 블록과 Future 객체의 catchError() 메서드를 함께 사용합니다.

예제 코드:

  • Future.catchError(): Future 체인에서 발생하는 비동기 예외를 처리합니다.
  • try-catch 블록: async-await를 사용하는 비동기 코드에서 await 중 발생하는 예외를 동기 코드처럼 처리할 수 있게 해줍니다.
  • finally 블록: try-catch 블록의 성공/실패 여부와 관계없이 항상 실행되어, 리소스 정리 등에 유용합니다.

10.7 Dart 패키지 사용하기 (Using Dart Packages)

2. 패키지 설치

  • as http는 http 패키지를 http라는 짧은 이름으로 사용하겠다는 별칭(alias)입니다. 이는 이름 충돌을 방지하고 코드를 간결하게 만듭니다.

10.10 Stream 소개 (Introduction to Stream)

  • Stream 은 Future와 마찬가지로 Dart에서 비동기 작업을 다루는 핵심 객체이지만, Future가 ‘미래의 단일 값’을 나타내는 반면, Stream은 ‘시간에 따라 여러 개의 값을 비동기적으로 전달하는 시퀀스’를 나타냅니다.
  • 즉, Stream은 데이터의 흐름(Flow) 을 표현하며, 작업이 완료될 때까지 기다리지 않고 데이터를 지속적으로 전달받아 처리할 수 있습니다.

Stream의 주요 특징:

  • 여러 개의 값: Future가 한 번의 값을 반환하는 것과 달리, Stream은 여러 개의 값을 순차적으로 전달할 수 있습니다.
  • 지속적인 데이터 흐름: 데이터가 한 번에 모두 전달되는 것이 아니라, 준비되는 대로 계속해서 전달됩니다.
  • 다양한 활용: 실시간 데이터(예: 채팅 메시지, 센서 데이터), 파일 읽기, 웹소켓 통신, 사용자 이벤트(예: 버튼 클릭) 등에 유용하게 사용됩니다.

10.11 Stream 사용하기: listen()

  • Stream에서 데이터를 받으려면 listen() 메서드를 사용해야 합니다. listen() 메서드는 Stream이 데이터를 전달할 때마다 호출되는 콜백 함수를 등록합니다.
  • Stream은 데이터를 준비되는 대로 ‘청크(chunk)’ 단위로 전달할 수 있습니다.

예제 코드:

  • listen() 메서드는 StreamSubscription 객체를 반환합니다. 이 객체를 사용하여 Stream의 수신을 제어(일시 중지, 재개, 취소)할 수 있습니다.

10.12 StreamTransformer로 데이터 변환 및 필터링

  • StreamTransformer는 Stream을 통해 전달되는 데이터를 변환하거나 필터링하여 새로운 Stream을 생성하는 데 사용됩니다.
  • 이는 Stream의 데이터를 가공하여 원하는 형태로 만들 때 매우 유용합니다.

예제 코드:

  • transform() 메서드를 사용하여 StreamTransformer를 직접 적용할 수도 있지만, where(), map(), expand() 등과 같은 편리한 메서드들이 이미 Stream 객체에 내장되어 있어 더 간결하게 사용할 수 있습니다.

10.13 async*와 yield로 Stream 생성하기

  • Stream을 직접 생성해야 하는 경우는 흔치 않지만, async* 함수와 yield 키워드를 사용하면 커스텀 Stream을 쉽게 만들 수 있습니다.
  • 이는 특정 로직에 따라 데이터를 순차적으로 방출해야 할 때 유용합니다.

Stream 생성 키워드:

  • async*: Stream을 반환하는 함수를 선언할 때 사용합니다. 일반 async 함수와 달리 Future 대신 Stream을 반환합니다.
  • yield: Stream에 데이터를 하나씩 방출(emit)할 때 사용합니다. return과 유사하지만, 함수를 종료하지 않고 다음 yield까지 실행을 일시 중지합니다.
  • yield*: 다른 Stream의 모든 데이터를 현재 Stream으로 전달할 때 사용합니다.

예제 코드:

  • await for 루프는 Stream의 모든 데이터를 순차적으로 처리할 때 listen() 메서드보다 더 간결하게 코드를 작성할 수 있도록 해줍니다.

10.14 StreamSubscription으로 Stream 제어하기

  • Stream.listen() 메서드는 StreamSubscription 객체를 반환합니다. 이 StreamSubscription 객체를 사용하면 Stream의 데이터 흐름을 제어(Control) 할 수 있습니다.
  • StreamSubscription을 통해 Stream을 일시 중지(pause), 재개(resume), 또는 완전히 취소(cancel)할 수 있습니다.

예제 코드:

  • listen() 메서드는 onError (오류 처리), onDone (Stream 완료 시), cancelOnError (오류 발생 시 자동 취소 여부)와 같은 추가적인 콜백 함수와 옵션을 받을 수 있습니다.

11. Modern Dart Features (Dart 3.0+)

11.1 레코드 소개 (Introduction to Records)

  • 레코드(Records) 는 Dart 3.0에서 도입된 기능으로, 여러 개의 값을 하나의 객체로 묶어서 반환하거나 전달할 수 있게 해주는 익명(anonymous)의 복합 타입 입니다.
  • 클래스를 정의하지 않고도 여러 값을 그룹화할 수 있어, 특히 함수에서 여러 값을 반환할 때 유용합니다.

레코드의 주요 특징:

  • 익명(Anonymous): 별도의 클래스 이름을 정의할 필요가 없습니다.
  • 복합 타입(Composite Type): 여러 타입의 값을 하나로 묶을 수 있습니다.
  • 불변(Immutable): 한 번 생성된 레코드의 값은 변경할 수 없습니다.
  • 타입 안전성: 각 필드의 타입이 명확하여 컴파일 시점에 오류를 방지합니다.

기본 사용법:

  • 레코드는 괄호 () 안에 쉼표로 구분된 값들을 나열하여 생성합니다.
  • 필드는 위치 기반(positional) 또는 이름 기반(named) 으로 정의할 수 있습니다.
  • 레코드는 함수에서 여러 값을 반환하거나, 간단한 데이터 구조를 임시로 만들 때 매우 유용합니다.

11.3 패턴 매칭: switch 표현식 (Pattern Matching: switch Expressions)

  • case 대신 =>를 사용하여 간결하게 표현합니다.
  • 모든 가능한 경우를 처리했는지 컴파일러가 검사하여 누락된 case를 방지합니다.

if-case 문의 특징:

  • if (값 case 패턴) 형식으로 사용합니다.
  • 값의 타입과 구조가 패턴과 일치하면 true가 되고, 패턴에서 값을 추출할 수 있습니다.

  3. 타입 안전성 향상: 예상치 못한 서브타입으로 인한 런타임 오류를 방지하고 코드의 안정성을 높입니다.

11.7 클래스 한정자 (Class Modifiers: base, final, interface, mixin)

  • Dart 3.0에서는 클래스의 상속(extends)과 구현(implements), 믹스인(with) 사용을 더 세밀하게 제어할 수 있는 클래스 한정자(Class Modifiers) 를 도입했습니다.
  • 이를 통해 클래스 설계자가 의도한 대로 클래스가 사용되도록 강제하여, 코드의 안정성과 예측 가능성을 높일 수 있습니다.

주요 클래스 한정자:

1. base 클래스

  • base 클래스는 자신을 상속(extends)하는 모든 클래스가 base 클래스여야 함을 강제합니다.
  • 외부 라이브러리에서 base 클래스를 implements할 수 없습니다. (같은 라이브러리 내에서는 가능)
  • 목적: 클래스 계층 구조 내에서만 상속을 허용하고, 외부에서 인터페이스로 구현되는 것을 막아 API의 안정성을 높입니다.

2. final 클래스

  • final 클래스는 자신을 상속(extends)하거나 구현(implements)하는 것을 모두 금지합니다.
  • 목적: 완전히 봉인된(closed) 클래스를 만들어, 더 이상 확장되거나 변경되지 않는 불변(immutable) 데이터 구조를 정의할 때 유용합니다.

3. interface 클래스

  • interface 클래스는 자신을 implements하는 것만 허용하고, 상속(extends)하는 것을 금지합니다.
  • 목적: 클래스를 순수하게 인터페이스로만 사용하도록 강제하여, API 계약을 명확히 합니다.

4. mixin class

  • mixin class는 일반 클래스처럼 인스턴스화되거나 상속될 수 있으면서, 동시에 믹스인(with)으로도 사용될 수 있는 클래스입니다.
  • 목적: 클래스와 믹스인의 기능을 모두 가지는 유연한 코드를 작성할 때 사용합니다.

12.3 Isolates (아이솔레이트)

1. Isolates란?

  • Isolates(아이솔레이트) 는 Dart의 동시성(concurrency) 모델로, 독립적인 메모리 공간에서 실행되는 작업 단위입니다.
  • 각 Isolate는 자체 메모리 힙을 가지므로 다른 Isolate와 메모리를 공유하지 않습니다.
  • 메모리를 공유하지 않기 때문에 락(lock)이나 뮤텍스(mutex) 없이도 안전하게 병렬 처리가 가능합니다.

Isolates의 특징:

  • 독립적인 메모리: 각 Isolate는 독립된 메모리 공간을 가지며 다른 Isolate와 메모리 공유 없음
  • 메시지 전달: SendPort와 ReceivePort를 통해 메시지를 주고받음
  • 진정한 병렬성: 멀티코어 CPU에서 실제 병렬 실행 가능
  • 안전성: 메모리 공유가 없어 데이터 경합(race condition) 문제 없음

Isolates vs Threads:

  • Thread: 메모리를 공유하여 빠르지만 동기화 필요
  • Isolate: 메모리를 공유하지 않아 안전하지만 메시지 전달 오버헤드 존재
